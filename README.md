# PakFasal

Smart farming companion app for Pakistan ‚Äî weather, marketplace, AI Q&A, sensor
data, learning videos, and a personalized crop calendar.

## Getting started

```bash
flutter pub get
```

### 1. Create your local config

All secrets and tunable URLs are injected at build time via `--dart-define`.
Nothing sensitive should ever live in source. Copy the example and fill it in:

```bash
cp config/app_config.example.json config/dev.json
```

Then edit `config/dev.json`. The file is **git-ignored**.

| Key                      | Required          | Description                                                                                |
|--------------------------|-------------------|--------------------------------------------------------------------------------------------|
| `APP_ENV`                | optional          | `dev` / `staging` / `prod` ‚Äî used as a label in logs / crash reports.                      |
| `YOUTUBE_API_KEY`        | optional          | YouTube Data API v3 key. If empty, the learning module silently shows local demo videos.   |
| `YOUTUBE_CHANNEL_ID`     | optional          | Restrict YouTube search to one channel.                                                    |
| `YOUTUBE_API_BASE_URL`   | optional          | Defaults to `https://www.googleapis.com/youtube/v3`.                                       |
| `WEATHER_API_BASE_URL`   | optional          | Defaults to `https://api.open-meteo.com/v1` (Open-Meteo is keyless).                       |

### 2. Run the app

Easiest ‚Äî use the helper script:

```powershell
# Windows
.\scripts\run_dev.ps1
```

```bash
# macOS / Linux
./scripts/run_dev.sh
```

Or run Flutter directly with the config file:

```bash
flutter run --dart-define-from-file=config/dev.json
```

### 3. Build a release APK

```powershell
.\scripts\run_dev.ps1 -Release -Build
```

```bash
./scripts/run_dev.sh --release --build
```

## Project structure

```
lib/
  main.dart                       # composition root, registers providers
  src/
    app.dart                      # MaterialApp + theme/locale wiring
    core/
      config/                     # AppConfig (build-time env values)
      localization/
      routing/                    # AppRoutes + AuthGate route guard
      theme/
      widgets/                    # shared UI (AuthGate, scaffolds, states)
    features/
      ai_query/
      auth/                       # AuthSessionController, login/signup
      crop_calendar/
      home/                       # dashboard
      learning/                   # YouTube videos, crop diseases catalog
      marketplace/
      onboarding/
      profile/
      sensor/
      splash/
      weather/                    # WeatherProvider (single source of truth)
```

Each feature follows `data/`, `domain/`, `presentation/{providers,screens,widgets}`.

## Configuration philosophy

* **Secrets**: only via `--dart-define` (or a git-ignored JSON loaded with
  `--dart-define-from-file`). Never inline.
* **Firebase config** (`lib/firebase_options.dart`) is auto-generated by the
  FlutterFire CLI. The keys it contains identify the project, not authenticate
  a user; they are restricted via Firebase Console (package name + SHA on
  Android, bundle id on iOS) and are safe to commit.
